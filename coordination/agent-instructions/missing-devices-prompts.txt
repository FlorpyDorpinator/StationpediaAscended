device-specific logic types like Mode, Setting, Ratio, Error, etc.

The Assembly-CSharp/ folder tells you WHAT those types actually mean.

AGENT 1 - Missing Batch 1 (29 devices)
================================================================================

Your task: Document logic descriptions for devices in missing-batch-1.json

**Your batch file:** `StationpediaPlus/coordination/outputs/missing-batch-1.json`

**Workflow:**
1. For each device in your batch:
   a. Find the device in `Stationeers/Stationpedia/Stationpedia.json` to get its LogicInsert array (list of logic types)
   b. For EVERY logic type, find the corresponding source file in `Assembly-CSharp/` 
   c. READ THE SOURCE CODE to understand what Mode/Setting/Ratio/Error values mean
   d. Look for enums, switch statements, or comments that explain the values

2. **Standard types** (use these descriptions as-is):
   - Power: "Returns 1 if device is powered."
   - On: "Device on/off state."
   - Lock: "Locks device when set to 1."
   - Pressure/Temperature/TotalMoles/RatioOxygen/etc: Use obvious gas descriptions

3. **Device-specific types** (MUST verify in source code):
   - Mode: Find the enum or switch statement that defines mode values
   - Setting: Find what Setting controls in the device's logic
   - Ratio: May be input ratio, output ratio, or fill ratio - check source
   - Error: Find error conditions in source code

**Output format:** Save to `StationpediaPlus/coordination/outputs/missing-batch-1-complete.json`

```json
[
  {
    "deviceKey": "ThingStructureExample",
    "displayName": "Example Device",
    "logicDescriptions": {
      "Power": { "dataType": "Boolean", "range": "0-1", "description": "Returns 1 if powered." },
      "Mode": { "dataType": "Integer", "range": "0-2", "description": "0=ModeA, 1=ModeB, 2=ModeC (from ExampleMode enum)." }
    }
  }
]
```

Work autonomously. Complete ALL devices. Report back when done.
